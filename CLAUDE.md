# Tabulator – Claude notes

Interactive table generation library. Vanilla JS, no framework runtime
dependencies. Docs/demos live at <http://tabulator.info>.

## Repo layout

- `src/js/core/` – `Tabulator.js` (orchestrator), `Module.js` /
  `CoreFeature.js` (base classes), `RowManager.js`, `ColumnManager.js`,
  plus subdirs for `cell/`, `column/`, `row/`, `rendering/`, `tools/`,
  `defaults/`.
- `src/js/core/builds/` – entry points for ESM, UMD, jQuery wrappers.
- `src/js/core/TabulatorFull.js` – the "everything bundled" class used
  in tests and demos.
- `src/js/core/modules/optional.js` – the registry of optional modules
  pulled in by `TabulatorFull`.
- `src/js/modules/<Name>/<Name>.js` – ~40 feature modules
  (`FrozenRows`, `Edit`, `Sort`, `Filter`, `Page`, etc.). Each extends
  `Module`, calls `registerTableOption` / `registerComponentFunction`
  in its constructor, and implements `initialize()`.
- `src/scss/` – `tabulator.scss` is the base; `themes/` contains
  Bootstrap, Bulma, Materialize, SemanticUI, modern, midnight, simple,
  site, site_dark.
- `dist/` – built artifacts (`js/tabulator.js`, `js/tabulator_esm.mjs`,
  `css/tabulator.css`, theme variants). Generated by `npm run build`.
- `test/unit/` – Jest specs, run in jsdom. `modules/<Name>.spec.js`
  mirrors the module layout. `setup.js` adds a few DOM helpers.
- `test/e2e/` – Playwright suite. Slow; needs a build first.

## Commands

- `npm run test:unit` – Jest only (fast, jsdom).
- `npm run test:e2e` – builds then runs Playwright.
- `npm test` – both.
- `npm run lint` – ESLint over `src/`, with `--fix`.
- `npm run build` – Rollup build (also runs lint via `prebuild`).
- `npm run dev` / `dev:css` / `dev:esm` / `dev:umd` / `dev:wrappers` –
  watch builds.

## Testing patterns

- Two flavours of unit test in `test/unit/modules/`:
  - **Pure unit** – import the module directly and stub
    `Module.prototype.registerTableOption` etc. so the module can be
    constructed without a real `Tabulator`. Heavy mocking of
    `table`, `rowManager`, `columnManager`.
  - **Integration** – `new TabulatorFull(element, options)` and
    `await new Promise(resolve => table.on("tableBuilt", resolve))`.
    Then drive the public API. See `Edit.spec.js`, `Page.spec.js`,
    `FrozenRowsIntegration.spec.js`.
- Pure-unit specs that overwrite globals on `document` (e.g.
  `document.createDocumentFragment = jest.fn(...)`) or
  `Module.prototype.*` **don't restore them per-test**, so adding
  `TabulatorFull` tests inside the same `describe` will break.
  Put integration tests in a separate file when this happens.
- jsdom doesn't compute layout – `offsetTop`/`offsetWidth` are mostly
  0. Test DOM structure (`previousSibling`, `children.length`,
  classes), not pixel positions. For layout bugs, assert on the
  structural fix.

## Build / rendering notes

- The build is Rollup-driven (`build/rollup.mjs`). SCSS is compiled
  with `sass` + `rollup-plugin-postcss`.
- Both `.tabulator-headers` and `.tabulator-frozen-rows-holder` are
  `display: inline-block`; structural separators (e.g. `<br>`) matter
  in the column-manager content area. See
  <https://github.com/tabulator-tables/tabulator/issues/4871>.

## Conventions

### Bug-fix issue links

When fixing a bug that's tracked in a GitHub issue, add a comment with
the full issue URL in two places:

1. **In the code**, next to the line(s) that implement the fix.
2. **In the test**, next to the regression test(s) that cover it.

Use the full URL (e.g.
`https://github.com/tabulator-tables/tabulator/issues/4871`) rather
than a bare `#4871`, so the link works from any context (IDE hover,
GitHub blame, etc.).

A future reader looking at the diff or `git blame` should be able to
find the original bug report without searching.
